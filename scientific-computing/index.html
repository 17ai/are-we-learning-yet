Scientific Computing

Several crates have populated scientific computing space especially regarding matrix operations and linear algebra. But it’s still too early to know which crates will emerge as central to a robust scientific computing ecosystem.

Given the value of iterative analysis in scientific computing, it’s worth checking out rusti REPL. There was some progress on a Jupyter backend for Rust but it seems to have stalled. You can also find a relevant rusti issue tracking this here.

Also, check out some of these resources:

  • A gemmed rabbit hole - concerning matrix multiplication
  • Linear Algebra in Rust - intro the the module that became rulinalg

collenchyma

An extensible, pluggable, backend-agnostic framework for parallel, high-performance computations on CUDA, OpenCL and common host CPU.

GSL

A rust binding for the GSL (the GNU scientific library)

scirust

A scientific computing library

numrs

A numerical computation library for Rust.

bio

This library provides rigorously tested implementations of many algorithms and data structures that are useful for bioinformatics.

matrixmultiply

General matrix multiplication of f32 and f64 matrices in Rust. Supports matrices with general strides.

rulinalg

A linear algebra library.

blas

The package provides an interface to the Basic Linear Algebra Subprograms.

lapack

The package provides an interface to the Linear Algebra PACKage.

euclid

Geometry primitives (basic linear algebra) for Rust

nalgebra

Linear algebra library for computer physics, computer graphics and general low-dimensional linear algebra for Rust.

cgmath

A linear algebra and mathematics library for computer graphics.
